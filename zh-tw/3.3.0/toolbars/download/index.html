啟動模擬器

中間的按鍵是用來開啟內建的模擬器(Simulator):

內建的模擬器是模擬使用者設計被載入 Ubique Panel 之後的顯示以及互動功能，使用者可以在個人電腦上以滑鼠操作設計出來的介面，模擬器會模擬完整圖像介面功能，並且支援所見及所得 (What-You-See-Is-What-You-Get) 的特色，模擬的圖像與 Ubique Panel 上一致(色差可能發生於使用者使用的螢幕以及 Ubique Panel 之間，其色域規格以 Ubique Panel 規格書定義為準)。

模擬器無法於個人電腦模擬 Ubique Panel 上的I/O、PWM、ADC等實體輸入輸出行為，但可以模擬 Ubique Panel 暫存器通訊協定。

最右邊的按鍵啟動屏通訊程式(Messenger)，目的是在個人電腦上與 Ubique Panel 或是模擬器進行通訊

如果打開對外通訊的屏通訊程式，會出現以下視窗:

在這視窗中，使用者可以透過個人電腦的 COM Port，連結 Ubique Panel，藉由暫存器通訊協定改變暫存器的內容值, 模擬由外部MCU透過 UART 傳遞數值給 Ubique Panel。

以下詳細說明與模擬器以及 Ubique Panel 通訊方法:

利用屏通訊程式和模擬器通訊

  • 首先用 ADE 開啟一個既有設計

    確認IOConfig頁面UART為啟動且UART鮑率正確且暫存器已選取

  • 到暫存器編輯視窗找到欲進行通訊的暫存器並勾選自動通訊上傳

  • 打開屏通訊程式視窗與模擬器，進行以下操作

    1. 設定Socket

    2. 點擊連接，連接成功會看到彈出信息框顯示成功

    3. 點擊連接確認，屏通訊程式會送出連接確認命令，若是模擬器端正常接收，則會回復連接確認命令。

    4. 連接正常，接收確認資訊ack

    5. 通訊指令

      選擇 Set Register 表示要寫入暫存器內容，並且設定暫存器序號以及要寫入的資料和型態，該暫存器ID以及資料型態必須與ADE設計專案內暫存器設定相符。

    6. 模擬器接收通訊指令且模擬器上會依通訊指令修改對應暫存器數值，若該暫存器有對應的顯示元件則模擬器進行相對應的顯示

屏通訊程式與Ubique Panel通訊

  • 首先利用ADE將既有的設計下載至博橡科技控制板上

  • 之後利用USB to TTL模組連接控制板RX、TX、GND與ADE所在電腦的USB

  • 連接完成後打開屏通訊程式視窗

    1. 設定Serial Port，利用Scan掃描更新串口，選擇正確的串口並設定鮑率

    2. 點擊Ack Test確認連接

    3. 連接正常，接收到ack訊息

    4. 設定並傳送通訊指令

  • 接下來即可在 Ubique Panel 上看到相對應的顯示

模擬器與Ubique Panel 通訊

目的為快速測試設計運作是正常

  • 首先，利用 ADE 準備兩個設計，一個負責傳送數值，一個負責接收數值

  • 將負責傳送數值的設計利用模擬器打開，負責接收數值的設計下載至博橡科技控制板

    ※注意!!確保兩個設計欲溝通的暫存器ID相同，且傳送方要勾選自動通訊上傳

    1. 設定Serial Port，選擇正確的串口並且設定好鮑率

    2. 利用元件改變暫存器內容，再透過UART傳送至博橡控制板

  • 接下來可以在 Ubique Panel 上看到相對應的顯示

Ubique Panel 和 MCU 通訊

  • 首先，準備一個 ADE Project並且設好暫存器。再參考博橡通訊協定寫出 Arduino 程式，將資料透過通訊傳給指定ID的暫存器。

  • 其次，連接博橡控制板與Arduino控制板的RX和TX

  • 將ADE Project與Arduino程式分別下載至兩個控制板，打開電源即可開始通訊。

博橡UART通訊協定

  • Command Format

    Command Lead Byte Type Byte Register ID Low Byte Register ID High Byte Tailing Bytes
    Set Register 0x11 0x00 0x00~0xFF 0x00~0xFF Data Payload
    Get Register 0x11 0x01 0x00~0xFF 0x00~0xFF N/A

  • Data Payload Format

    Payload Type ID Byte Length Low Byte Length High Byte Tailing Bytes
    Boolean 0x00 0x01 0x00 Data Bytes
    String 0x01 0x01~0xFF 0x00~0xFF Data Bytes
    Integer 0x02 0x01/02/04/08 0x00 Data Bytes
    Unsigned Integer 0x03 0x01/02/04/08 0x00 Data Bytes
    Floating Point 0x04 0x04/08 0x00 Data Bytes

  • Data Bytes are of the Little-Endian Format
